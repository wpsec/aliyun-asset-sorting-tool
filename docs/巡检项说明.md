# 阿里云资产梳理巡检项说明

本文档说明 `scripts/aliyun_asset_inventory.py` 当前内置的日常运维巡检项。所有巡检只读取资产元数据、配置摘要和云监控指标，不修改云资源。

## 结果字段

- `风险等级`：`高`、`中`、`低`。
- `整改优先级`：`P0-高危暴露`、`P1-闲置计费资源`、`P2-身份风险`、`P2-中风险配置`、`P3-一般优化`。
- `分类`：`高风险暴露`、`闲置资源`、`未使用配置`、`疑似闲置`。
- `计费属性`：`持续计费`、`用量计费`、`通常不直接计费`、`疑似持续计费`、`需账单确认`。
- `计费说明`：说明该资源为什么可能产生费用，或为什么更偏配置/安全治理。
- `证据`：用于人工复核的元数据，不包含 AK/SK、对象内容、日志正文或密钥明文。

`闲置资源` 工作表会结合计费属性排序和复核。云盘、EIP、NAT、VPN、负载均衡等属于优先成本复核对象；VPC、交换机、服务器组等通常不直接计费，主要用于发现可能可清理的网络配置。

## 白名单

在 `config/checks.example.json` 的 `白名单` 中配置：

- `资源ID`：按资源 ID 跳过巡检发现。
- `资源组ID`：按资源组 ID 跳过巡检发现。
- `标签`：按标签跳过巡检发现，`值` 支持 `*`。

典型场景：

- 保留盘、模板安全组、预留 EIP。
- 审计保留 Bucket。
- 特殊演练或灾备环境。

## 静态巡检项

| 检查项ID | 风险 | 分类 | 说明 |
| --- | --- | --- | --- |
| `ecs_unused_disk` | 中 | 闲置资源 | 云盘未挂载 ECS 实例。 |
| `ecs_stopped_instance` | 低 | 闲置资源 | ECS 实例处于停止状态。 |
| `ecs_unused_security_group` | 低 | 未使用配置 | 安全组未绑定 ECS 实例或弹性网卡。 |
| `ecs_high_risk_security_group_rule` | 高 | 高风险暴露 | 安全组向 `0.0.0.0/0` 或 `::/0` 开放 22、3389、3306、5432、6379、9200、9300、27017 等端口型高危协议；可信内网段、堡垒机出口和固定 IP 白名单不纳入该项。 |
| `ecs_disk_without_snapshot_policy` | 低 | 未使用配置 | 云盘未发现自动快照策略关联。 |
| `vpc_empty_vpc` | 低 | 闲置资源 | VPC 未发现 ECS 实例或弹性网卡直接绑定，不等同于确认可删除。 |
| `vpc_empty_vswitch` | 低 | 闲置资源 | 交换机未发现 ECS 实例或弹性网卡直接绑定，不等同于确认可删除。 |
| `vpc_unused_eip` | 中 | 闲置资源 | EIP 未绑定云资源。 |
| `vpc_nat_without_rules` | 低 | 闲置资源 | NAT 网关未发现 SNAT 或 DNAT 规则。 |
| `vpc_vpn_without_connection` | 低 | 闲置资源 | VPN 网关未发现 IPsec 连接或 SSL-VPN 服务端配置。 |
| `slb_without_listener` | 中 | 闲置资源 | 负载均衡未发现监听。 |
| `slb_server_group_without_backend` | 中 | 闲置资源 | ALB/NLB 服务器组未发现后端服务器。 |

| `ram_access_key_no_usage_record` | 低/中 | 未使用配置 | Active AK 无使用记录（`GetAccessKeyLastUsed` 未返回数据），无法确认是否仍在使用，需人工复核；绑定高危策略时严重度升级为中。 |
| `ram_root_access_key` | 高 | 高风险暴露 | Root 账号存在 Active 状态 AccessKey，零阈值即报。 |
| `ram_access_key_rotation` | 中 | 身份安全 | Active AK 创建超过轮转阈值天数（默认 365）未轮转。 |
| `ram_inactive_access_key_cleanup` | 低 | 未使用配置 | Inactive AK 残留超过清理阈值天数（默认 90），提示删除。 |
| `ram_user_without_mfa` | 中 | 未使用配置 | RAM 用户未发现 MFA 绑定信息。 |
| `ram_empty_group` | 低 | 未使用配置 | RAM 用户组未发现成员。 |
| `oss_bucket_without_encryption` | 中 | 未使用配置 | Bucket 未发现服务端加密配置。 |
| `oss_bucket_public_access` | 高/中 | 高风险暴露 | Bucket ACL 或 Policy 存在公开访问风险。 |
| `oss_bucket_without_lifecycle` | 低 | 未使用配置 | Bucket 未发现生命周期规则。 |

| `redis_public_endpoint` | 中 | 高风险暴露 | Redis/Tair 实例存在公网连接地址。 |

## 指标类巡检项

指标类巡检默认关闭。启用后会通过 `cms:DescribeMetricData` 做近 7/14/30 天趋势判断，仅输出“疑似闲置”，不等同于可删除。

| 检查项ID | 默认阈值 | 说明 |
| --- | --- | --- |
| `metric_idle_ecs` | CPU 平均值不超过 3%，公网出方向平均值不超过 1024 Bps | ECS 疑似低使用率。 |
| `metric_idle_eip` | 平均流量不超过 1024 Bps | EIP 疑似低使用率。 |
| `metric_idle_slb` | 平均 QPS 不超过 0.1 | SLB 疑似低使用率。 |
| `metric_idle_rds` | 平均连接数不超过 1 | RDS 疑似低使用率。 |
| `metric_idle_redis` | 平均 QPS 不超过 0.1 | Redis/Tair 疑似低使用率。 |

指标阈值可在 `config/checks.example.json` 的 `指标巡检.阈值` 中调整。
需要排查低使用率时，可以将 `指标巡检.启用` 设置为 `true`。资源量较大时建议保持关闭，避免增加大量云监控 API 调用。

## 误报复核建议

- 闲置资源：先确认负责人、资源组、标签和近期变更记录。
- 高风险暴露：先确认公网入口是否有上游防护、ACL、WAF 或堡垒机限制；可信内网段和固定白名单属于正常管理面访问，不应与公网开放等价。
- 未使用配置：确认是否为模板、预留、灾备或审计保留。
- 疑似闲置：至少结合业务日历和告警记录人工确认，不建议直接释放。
