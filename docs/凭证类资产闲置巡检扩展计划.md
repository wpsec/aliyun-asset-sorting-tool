# 凭证类资产闲置巡检扩展计划

## 背景

AK/SK 闲置巡检增强已完成（分级闲置、Root AK、轮转、Inactive 清理）。同类模式——"凭证/密钥长期未使用或未轮转"——在阿里云其他服务中同样存在，需要统一扩展覆盖。

## 扩展方向概览

从 AK/SK 的"闲置/轮转/残留"模式泛化到全栈凭证类资产，按安全权重和实施难度排序。

## P0 — 高安全权重 + 低实施难度

### 1. KMS 密钥闲置 & 未轮转

| 维度 | 说明 |
| --- | --- |
| 资产类型 | KMS Customer Master Key（对称/非对称/HMAC） |
| 风险 | KMS 密钥是数据加密的根信任，泄露/失控影响比 AK 更深远；Disabled 密钥残留 = 加密数据无法解密风险 + 凭证泄露面 |
| 数据获取 | `kms:ListKeys` 返回密钥 ID 列表；`kms:DescribeKey` 返回 `KeyState`(Enabled/Disabled/PendingDeletion)、`CreationDate`、`LastUsageDate`、`AutomaticRotation`(是否自动轮转)、`KeyUsage` |
| 采集位置 | 新增 `collect_kms_details`，在 `collect_detailed_assets` 中调用 |
| DetailedAssets 新增字段 | `kms_keys: list[dict]` |

巡检项设计：

| 检查项ID | 严重度 | 分类 | 条件 | 说明 |
| --- | --- | --- | --- | --- |
| `kms_stale_key` | 低/中/高 | 未使用配置 | Enabled 密钥闲置天数分级：30d(低) / 90d(中) / >=180d(高)，参考 AK 分级阈值 | KMS 密钥长期未使用 |
| `kms_key_without_rotation` | 中 | 身份安全 | Enabled 密钥创建超过阈值天数且 `AutomaticRotation` 未开启 | 密钥未开启自动轮转 |
| `kms_disabled_key_cleanup` | 低 | 未使用配置 | Disabled 密钥超过阈值天数未删除，参考 Inactive AK 清理 | Disabled 密钥长期残留 |
| `kms_pending_deletion_key` | 低 | 未使用配置 | PendingDeletion 状态密钥 | 密钥等待删除（7-30 天窗口期），提示确认 |

配置扩展：

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `stale_kms_key_warn_days` | int | 30 | KMS 密钥闲置低风险阈值 |
| `stale_kms_key_days` | int | 90 | KMS 密钥闲置中风险阈值 |
| `stale_kms_key_severe_days` | int | 180 | KMS 密钥闲置高风险阈值 |
| `kms_key_rotation_days` | int | 365 | 密钥未轮转阈值 |
| `kms_disabled_key_cleanup_days` | int | 90 | Disabled 密钥清理阈值 |

### 2. SSL/TLS 证书过期预警

| 维度 | 说明 |
| --- | --- |
| 资产类型 | 数字证书管理服务（CAS）中的用户证书 |
| 链式影响 | CDN/OSS/SLB/ALB 绑定证书过期会导致服务中断或浏览器告警，比 AK 闲置的后果更直接 |
| 数据获取 | `cas:DescribeUserCertificateList` 返回证书列表，含 `StartDate`、`EndDate`、`Status`(ISSUED/EXPIRED/REVOKED)、`CertType`(DV/OV/EV)、绑定域名 |
| 采集位置 | 新增 `collect_ssl_certificate_details`，在 `collect_detailed_assets` 中调用 |
| DetailedAssets 新增字段 | `ssl_certificates: list[dict]` |

巡检项设计：

| 检查项ID | 严重度 | 分类 | 条件 | 说明 |
| --- | --- | --- | --- | --- |
| `ssl_certificate_expiring` | 高 | 高风险暴露 | 距 EndDate <= 30 天且状态为 ISSUED | 证书即将过期，业务即将中断 |
| `ssl_certificate_expired` | 高 | 高风险暴露 | EndDate < now 且状态仍为 ISSUED（API 可能已标记 EXPIRED） | 证书已过期 |
| `ssl_certificate_long_validity` | 低 | 未使用配置 | 证书有效期超过 365 天（DV 证书通常有效期短，OV/EV 可能较长） | 证书有效期过长，建议缩短 |

配置扩展：

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `ssl_expiring_warn_days` | int | 30 | 证书即将过期预警天数 |
| `ssl_long_validity_days` | int | 365 | 证书有效期过长阈值 |

## P1 — 中安全权重 + 低实施难度

### 3. ECS SSH 密钥对未绑定实例

| 维度 | 说明 |
| --- | --- |
| 资产类型 | ECS SSH 密钥对 |
| 风险 | 残留密钥对 = 潜在横向渗透通道，虽不直接计费 |
| 数据获取 | `ecs:DescribeKeyPairs` 返回密钥对列表，含 `KeyPairName`；可通过 `InstanceCount` 或遍历 ECS 实例的 `SSHKeyPairName` 字段判断绑定关系 |
| 采集位置 | 在现有 ECS 详情补采中扩展，或在 `collect_ram_details` 附近新增 |
| DetailedAssets 新增字段 | `ecs_ssh_key_pairs: list[dict]` |

巡检项设计：

| 检查项ID | 严重度 | 分类 | 条件 | 说明 |
| --- | --- | --- | --- | --- |
| `ecs_unused_ssh_key_pair` | 低 | 未使用配置 | 密钥对未绑定任何 ECS 实例 | SSH 密钥对未绑定实例 |

## P2 — 需交叉比对或策略解析

### 4. RAM 自定义策略孤立项

| 维度 | 说明 |
| --- | --- |
| 资产类型 | RAM 自定义策略（Custom Policy） |
| 风险 | 未绑定任何身份的策略是配置噪音，存在误授权风险（将来可能被意外绑定） |
| 数据获取 | `ram:ListPolicies`（--PolicyType Custom）返回策略列表；交叉比对 `ListPoliciesForUser` / `ListPoliciesForGroup` / `ListPoliciesForRole` 判断是否有主体绑定 |
| 采集位置 | 在 `collect_ram_details` 中扩展 |
| DetailedAssets 新增字段 | `ram_custom_policies: list[dict]` |

巡检项设计：

| 检查项ID | 严重度 | 分类 | 条件 | 说明 |
| --- | --- | --- | --- | --- |
| `ram_orphan_custom_policy` | 低 | 未使用配置 | 自定义策略未绑定任何 RAM 用户/组/角色 | 自定义策略无授权主体 |

### 5. RAM 角色跨账号信任闲置

| 维度 | 说明 |
| --- | --- |
| 资产类型 | RAM 角色（含跨账号信任策略的角色） |
| 风险 | 跨账号信任 = 安全边界开放，长期不被 Assume = 应收紧信任范围 |
| 数据获取 | `ram:ListRoles` 返回角色列表；`AssumeRolePolicyDocument` 中 `Principal` 含外部账号 ARN 时标记为跨账号信任；需结合 ActionTrail 日志判断最近 Assume 时间（非本项目采集范围，可标记为"无法判断使用状态"） |
| 采集位置 | 在 `collect_ram_details` 中扩展 |
| DetailedAssets 新增字段 | `ram_roles: list[dict]` |

巡检项设计：

| 检查项ID | 严重度 | 分类 | 条件 | 说明 |
| --- | --- | --- | --- | --- |
| `ram_role_external_trust` | 中 | 高风险暴露 | 角色信任策略含外部账号 Principal | 角色允许外部账号 Assume，需确认必要性 |

### 6. VPN SSL 客户端证书过期

| 维度 | 说明 |
| --- | --- |
| 资产类型 | SSL-VPN 客户端证书 |
| 风险 | 过期证书用户无法连接但证书仍存在，残留 = 管理噪音 |
| 数据获取 | `vpc:DescribeSslVpnClientCerts` 返回证书列表，含 `Status`(Normal/Expired)、`CreateTime`、`ExpireTime` |
| 采集位置 | 在现有 VPC 详情补采中扩展 |

巡检项设计：

| 检查项ID | 严重度 | 分类 | 条件 | 说明 |
| --- | --- | --- | --- | --- |
| `vpc_ssl_vpn_cert_expired` | 低 | 未使用配置 | SSL VPN 客户端证书已过期 | SSL VPN 客户端证书过期，建议删除 |

## P3 — 需额外数据源或指标采集

### 7. RDS 数据库账号闲置（P3 — 需审计日志）

阿里云 RDS API 不直接返回账号最后登录时间，需结合 ActionTrail 或 SQL 审计日志间接判断，实施难度高，暂不纳入。

### 8. 函数计算长期无调用（P3 — 偏业务指标）

FC 调用率属于业务层面低使用率，和现有 `metric_idle_*` 模式类似，需要额外指标采集，且不在当前"凭证类资产"范畴内，暂不纳入。

## 实施路线图

| 阶段 | 巡检项 | 涉及文件 | API 调用增量 |
| --- | --- | --- | --- |
| P0-1 | KMS 密钥 4 项 | `scripts/aliyun_asset_inventory.py` (采集+normalize) / `checks/kms.py` (新增) / `checks/models.py` / `checks/runner.py` / `config/checks.example.json` / `tests/test_checks.py` / `docs/巡检项说明.md` | 单订阅约 +2N 次（N=密钥数量，DescribeKey 逐个） |
| P0-2 | SSL 证书 3 项 | 同上结构，`checks/ssl_cert.py` (新增) | 单订阅约 +1 次（DescribeUserCertificateList 一次性返回） |
| P1-1 | SSH 密钥对 1 项 | 在 ECS 采集和 `checks/ecs.py` 中扩展 | 单订阅约 +R 次（R=ECS 地域数，DescribeKeyPairs 每地域一次） |
| P2-1 | RAM 自定义策略孤立 | 在 RAM 采集和 `checks/ram.py` 中扩展 | 单订阅约 +3M 次（M=自定义策略数，需交叉比对 User/Group/Role） |
| P2-2 | RAM 角色跨账号信任 | 在 RAM 采集和 `checks/ram.py` 中扩展 | 单订阅约 +1 次（ListRoles） + 逐个解析 AssumeRolePolicyDocument |
| P2-3 | VPN SSL 客户端证书过期 | 在 VPC 采集和 `checks/vpc.py` 中扩展 | 单订阅约 +R 次（R=VPN 地域数） |

## 注意事项

1. **KMS DescribeKey 逐个调用**：`ListKeys` 仅返回 KeyId，需逐个 `DescribeKey` 获取完整元数据。密钥数量多时 API 调用增量较大，可考虑仅在启用 KMS 相关巡检项时才采集。
2. **SSL 证书地域**：CAS 是全局服务，证书查询不按地域，一次调用即可。
3. **RAM 策略交叉比对**：P2-1 需对每个自定义策略分别查询 User/Group/Role 绑定关系，调用量与策略数成正比。策略数多的账号需评估 API 限流风险。
4. **证书过期链式影响**：巡检只检查证书本身状态，不自动关联 CDN/OSS/SLB 绑定关系。后续可扩展"证书即将过期 + 当前被 SLB/CDN 使用"的组合巡检。
5. **KMS Disabled 密钥的特殊性**：Disabled 密钥仍可解密已有数据但不能加密新数据，残留 Disabled 密钥不等同于可安全删除——需要确认是否有依赖该密钥的存量加密数据。
