# PlantUML 拓扑导出说明

## 设计决策

- 拓扑输出采用 PlantUML 语法嵌入 Markdown 文件，一个订阅下的每个 VPC 单独输出一份拓扑图（`<region>__<vpc-id>.md`），同一订阅内可能有多个 VPC 因此会有多份拓扑图。
- 只根据显式字段和确定性映射生成资源关系，不按名称、同地域或同标签猜测依赖。
- 目标是"资源关系拓扑"，不是"真实访问路径图"或"流量路径图"。
- VPC 为顶层分组，VSwitch 为子分组，其余资源根据 VSwitch/VPC 归属放置。
- 无法确认 VPC 归属的资源写入 `unassigned.md`。

| VSwitch    | rectangle     | 绿 #C8E6C9    | VPC 内子分组          |
| ECS        | node          | #LightBlue    | 可挂到 VSwitch 或 VPC |
| ENI        | node          | #LightCyan    | 可挂到 VSwitch 或 VPC |
| EIP        | node          | #Magenta      | 公网资源，独立节点     |
| NAT        | rectangle     | #BBDEFB       | VPC 内独立节点        |
| VPN Gateway | rectangle    | #BBDEFB       | VPC 内独立节点        |
| VPN Connection | rectangle | #BBDEFB       | 挂到 VPN Gateway      |
| SSL-VPN    | rectangle     | #BBDEFB       | 挂到 VPN Gateway      |
| SLB/ALB/NLB | rectangle    | #BBDEFB       | VPC 内独立节点        |

## 连线规则

- **实线箭头 `-->`**：资源归属关系和业务依赖
  - VPC --> VSwitch : "包含"
  - EIP --> ECS : "绑定"
  - LB --> Listener : "监听"
  - Listener --> ServerGroup : "转发"
  - VSwitch anchor --> VSwitch 内资源

- **虚线箭头 `..>`**：附属/管理关系
  - VPN Gateway ..> VPN Connection
  - VPN Gateway ..> SSL-VPN
  - ServerGroup ..> Backend Server

- **连线标注**：关键关系附带说明文本（绑定、监听、转发、后端等）

## 第一版边界

覆盖的资源类型：VPC、VSwitch、ECS、ENI、EIP、NAT、VPN Gateway、VPN Connection、SSL-VPN、SLB/ALB/NLB、Listener、ServerGroup、Backend Server、RDS、Redis/Tair。

不包含：磁盘、OSS、RAM、KMS、SLS、CEN、DNS/PrivateZone、ACK Pod/Service、CLB 后端。

## 渲染方式

PlantUML 代码块嵌入 `.md` 文件后，可通过以下方式渲染：

1. **PlantUML Online Server**：将 `@startuml/@enduml` 之间的代码复制到 https://www.plantuml.com/plantuml/uml 直接生成 PNG/SVG。
2. **VS Code 插件**：安装 PlantUML 插件（如 `jebbs.plantuml`），在 VS Code 中直接预览。
3. **IntelliJ 插件**：安装 PlantUML integration 插件。
4. **命令行工具**：安装 `plantuml`（需 Java 运行环境），执行 `plantuml topology/*.md` 生成图片。

## 关键接口查询失败

当拓扑相关 API 查询失败时，仍会输出部分拓扑，并在对应 Markdown 文件顶部标注"部分关系缺失"，列出失败的服务、API 和错误信息。
