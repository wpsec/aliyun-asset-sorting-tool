# list 资产梳理策略包说明

更新时间：2026-05-06

## 1. 策略定位

这组策略用于阿里云云资产梳理，只授予资产元数据查看权限。

目标边界：

- 允许枚举云资源、标签、地域、规格、网络关系、实例状态、基础安全配置、监控指标和安全产品资产摘要。
- 不允许创建、修改、删除、启停、授权、执行命令、生成凭据、读取密钥明文。
- 不允许读取 OSS Object 内容或对象列表。
- 不允许读取 SLS 日志正文。
- 不允许读取 ECS UserData、云助手执行结果、实例控制台输出、实例截图。
- 不允许读取 ACK kubeconfig、节点接入脚本、镜像仓库登录 Token。

## 2. 为什么拆成 5 个策略

本策略包不使用 Action 通配符。为了避免单个 RAM 自定义策略内容过长，将资产梳理权限按资产域拆分。

文件命名规则采用 `list-服务码-服务码.json`，便于直接看出每个策略覆盖哪些云产品。

需要把下面 5 个策略全部绑定到同一个资产梳理 RAM 用户或 RAM 角色：

| 策略文件 | 建议策略名 | 覆盖范围 |
| --- | --- | --- |
| `policies/list-resourcecenter-resourcemanager-resourcesharing-tag-ram-config-actiontrail.json` | `list-resourcecenter-resourcemanager-resourcesharing-tag-ram-config-actiontrail` | 资源中心、资源目录、资源共享、标签、RAM、配置审计、操作审计 |
| `policies/list-ecs-cs-cr-eci.json` | `list-ecs-cs-cr-eci` | ECS、ACK、ACR、ECI |
| `policies/list-vpc-slb-alb-nlb-cen-ga-cdn-dcdn-dns-pvtz.json` | `list-vpc-slb-alb-nlb-cen-ga-cdn-dcdn-dns-pvtz` | VPC、SLB、ALB、NLB、CEN、GA、CDN、DCDN、DNS、PrivateZone |
| `policies/list-rds-polardb-kvstore-dds-adb-hbase-oss-nas-ebs.json` | `list-rds-polardb-kvstore-dds-adb-hbase-oss-nas-ebs` | RDS、PolarDB、Tair/Redis、MongoDB、ADB、HBase、OSS、NAS、EBS |
| `policies/list-cms-arms-log-kms-sas-cloudfw-waf-bastionhost-pam.json` | `list-cms-arms-log-kms-sas-cloudfw-waf-bastionhost-pam` | 云监控、ARMS、SLS 元数据、KMS 元数据、云安全中心、云防火墙、WAF、堡垒机 |

## 3. 当前安全收敛

当前版本做了这些收敛：

- `Action` 不使用 `*` 或 `?` 通配符。
- 不包含 `Effect: Deny`，避免 RAM 控制台摘要因为显式拒绝动作显示 `write`。
- 不包含 `Create`、`Update`、`Modify`、`Delete`、`Attach`、`Detach`、`Put`、`Post`、`Run`、`Start`、`Stop`、`Enable`、`Disable` 等变更类动作。
- OSS 只保留 Bucket 级元数据动作，不包含 Object 内容读取和 Object 列表。
- SLS 只保留 Project、Logstore、索引、机器组、告警、仪表盘等元数据动作，不包含日志查询和拉取。
- ARMS 不授予 `GetPrometheusInstance`、`GetRumAppInfo`、联系人、Webhook、OnCall 和指标查询动作，避免返回 Token、个人信息或非资产数据。
- KMS 只保留密钥、凭据、策略、实例等 `Describe/List` 元数据动作，不包含公钥导出、解密、数据密钥生成、凭据值读取。
- ECI 已移除 `DescribeContainerLog`，避免读取容器日志正文。
- ACK 不包含 kubeconfig 和节点接入脚本相关动作。
- ACR 不包含 `GetAuthorizationToken`。

`Resource` 仍保留 `"*"`。这是资产梳理类策略的正常取舍：很多列表、搜索、跨地域和账号级 API 不支持在调用前限定具体资源 ARN；如果强行收窄，资产清单会明显漏项。

## 4. 使用建议

绑定时建议：

- 只给资产梳理 RAM 用户或角色绑定这 5 个自定义策略。
- 不要叠加 `AdministratorAccess`。
- 不要叠加任何 `*FullAccess`。
- 不要叠加宽泛系统只读策略，避免额外读取业务数据或敏感元数据。
- 不要使用创建策略的管理员 AK/SK 执行资产梳理。
- AK/SK 使用完成后按周期轮换；长期任务优先使用 RAM Role 或受控临时凭证。

## 5. 控制台验证重点

在 RAM 控制台查看每个策略摘要时，建议重点确认：

- “允许”区域不应出现 `write`。
- 不应出现“显式拒绝”区域。
- 不应出现“未识别服务”。
- 不应出现“无效授权”。
- 访问级别应只包含 `read` / `list`。

如果某个服务仍显示 `write`，不要直接加 Deny；应定位对应 Action，把 RAM 判定为 `write` 的动作从 Allow 中移除或替换成更窄的只读动作。

## 6. 正向验证

使用资产梳理 AK/SK 验证这些元数据查询应成功：

```bash
aliyun resourcecenter SearchResources
aliyun ecs DescribeInstances --RegionId cn-shanghai
aliyun vpc DescribeVpcs --RegionId cn-shanghai
aliyun rds DescribeDBInstances --RegionId cn-shanghai
aliyun oss ListBuckets
```

根据实际资产情况，也可以补充：

```bash
aliyun cs DescribeClusters --RegionId cn-shanghai
aliyun cbn DescribeCens
aliyun cms DescribeMetricMetaList
```

也可以使用仓库里的脚本做验证、原始 CSV 导出和 Excel 报告导出。脚本只使用 `~/.aliyun/config.json` 里已经配置好的 profile 名，不读取、不输出 AK/SK。
建议在项目根目录使用 `aliyun_asset_inventory.py` 入口执行。默认输出目录固定为项目根目录的 `outputs/`，即使从 `scripts/` 目录执行也不会写到 `scripts/outputs/`。

如果还没有配置 profile，先用交互式方式配置，避免把 AK/SK 写进命令历史或脚本：

```bash
aliyun configure --mode AK --profile asset-inventory
```

只做权限验证：

```bash
python3 aliyun_asset_inventory.py --profile asset-inventory --verify-only
```

多个订阅或环境可以一次输入多个 profile：

```bash
python3 aliyun_asset_inventory.py --profiles dev,prod,sec --verify-only
```

也可以直接进入交互式输入：

```bash
python3 aliyun_asset_inventory.py --interactive --verify-only
```

导出前 20 条资产做冒烟测试：

```bash
python3 aliyun_asset_inventory.py --profile asset-inventory --verify --limit 20 --output-dir outputs/aliyun-assets-smoke
```

默认输出两类文件：

- `raw-resourcecenter.csv`：资源中心原始清单，适合留作底稿。
- `asset-report.xlsx`：可交付报告，按工作表拆分资产，并补充 ECS、VPC、SLB/ALB/NLB、RAM、OSS、云安全中心等巡检和漏洞修复命令所需只读元数据。

`asset-report.xlsx` 主要工作表：

- `报告说明`
- `总览统计`
- `采集问题`
- `巡检总览`
- `巡检明细`
- `闲置资源`
- `未使用配置`
- `高风险暴露`
- `指标疑似闲置`
- `ECS云服务器`
- `ECS磁盘`
- `ECS安全组`
- `ECS快照`
- `网络与负载均衡`
- `数据库`
- `存储`
- `容器`
- `漏洞修复命令`
- `安全监控`
- `全部资源`

导出多个订阅的全量资产清单，一个订阅一个目录，并额外生成合并 CSV 和合并 Excel 报告：

```bash
python3 aliyun_asset_inventory.py --profiles dev,prod,sec --output-dir outputs/aliyun-assets
```

输出目录示例：

```text
outputs/aliyun-assets/
  dev/raw-resourcecenter.csv
  dev/findings.csv
  dev/asset-report.xlsx
  prod/raw-resourcecenter.csv
  prod/findings.csv
  prod/asset-report.xlsx
  sec/raw-resourcecenter.csv
  sec/findings.csv
  sec/asset-report.xlsx
  all-subscriptions.csv
  all-subscriptions-findings.csv
  all-subscriptions-report.xlsx
```

如果是资源目录多账号梳理，使用资源目录 ID、Root 资源夹 ID、资源夹 ID 或成员账号 ID 作为 scope。所有 profile 使用同一个 scope：

```bash
python3 aliyun_asset_inventory.py --profiles dev,prod --multi-account-scope <rd-or-folder-or-account-id> --output-dir outputs/aliyun-assets-multi-account
```

不同 profile 使用不同输出目录名或 scope 时，可以用 `--subscription`，格式是 `profile,目录名,scope`：

```bash
python3 aliyun_asset_inventory.py \
  --subscription dev,dev-cn-shanghai \
  --subscription prod,prod-main,<rd-or-folder-or-account-id> \
  --output-dir outputs/aliyun-assets
```

## 7. 日常运维巡检权限说明

脚本默认会执行静态巡检，只读取资产元数据和配置摘要，不读取业务数据，不执行修复动作。

第一版巡检依赖的只读接口主要包括：

- ECS：`DescribeInstances`、`DescribeDisks`、`DescribeSecurityGroups`、`DescribeSecurityGroupAttribute`、`DescribeNetworkInterfaces`。
- VPC：`DescribeEipAddresses`、`DescribeNatGateways`、`DescribeSnatTableEntries`、`DescribeForwardTableEntries`、`DescribeVpnGateways`、`DescribeVpnConnections`、`DescribeSslVpnServers`。
- SLB/ALB/NLB：负载均衡、监听、服务器组和后端服务器的 `Describe/List/Get` 元数据接口。
- RAM：`ListUsers`、`ListAccessKeys`、`GetAccessKeyLastUsed`、`GetUserMFAInfo`、`ListGroups`、`ListUsersForGroup`。
- OSS：`GetBucketAcl`、`GetBucketEncryption`、`GetBucketPolicyStatus`、`GetBucketPublicAccessBlock`。
- ECS 自动快照：`DescribeAutoSnapshotPolicyAssociations`。
- 数据库：`DescribeDBInstances`、`DescribeDBInstanceNetInfo`、Redis/Tair 的实例和连接地址元数据接口。
- 云监控：`cms:DescribeMetricData`，用于可选的 ECS、EIP、SLB、RDS、Redis/Tair 疑似闲置指标判断。

这些接口均已在 5 份 list 策略中按明确 Action 授权，不需要新增写权限。巡检不会调用 OSS Object 列表或对象读取，不会读取 SLS 日志正文，不会读取 KMS 凭据值，不会获取 ACK kubeconfig，也不会获取镜像仓库登录 Token。

如果只想导出资产、不执行巡检，可以使用：

```bash
python3 aliyun_asset_inventory.py --profile asset-inventory --no-checks
```

巡检阈值可以通过 `config/checks.example.json` 调整，例如高危端口列表、AK 未使用天数和启用/禁用具体检查项。示例配置使用中文字段，脚本也兼容旧版英文字段。
指标巡检默认关闭，因为它会增加大量云监控 API 调用。需要排查低使用率时，可以将 `指标巡检.启用` 设置为 `true`。

巡检结果会同时输出：

- `findings.csv`：当前订阅巡检发现。
- `all-subscriptions-findings.csv`：多订阅合并巡检发现。

如果要复用已有 `raw-resourcecenter.csv`，只重新生成巡检结果和报告，可以使用：

```bash
python3 aliyun_asset_inventory.py --profile asset-inventory --checks-only --output-dir outputs/aliyun-assets
```

## 8. 反向验证

下面这些动作不应被允许：

```bash
aliyun kms GetSecretValue --SecretName example-secret
aliyun ecs DescribeInvocationResults --RegionId cn-shanghai
aliyun cs DescribeClusterUserKubeconfig --ClusterId example-cluster-id
```

验证时不要使用真实业务密钥、真实业务对象或真实敏感数据。

## 9. 后续补权限原则

资产梳理脚本如果遇到 `Forbidden`：

- 先确认接口是否只返回资产元数据。
- 只补充明确的 `Describe`、`List`、少量必要 `Get` 动作。
- 不使用 Action 通配符。
- 不补充数据面读取动作，例如对象内容、日志正文、密钥明文、Token、kubeconfig。
- 不补充任何变更、授权、执行、启停类动作。
